# Automatic Annotation Pipeline (GroundingDINO + Autodistill)

This project automatically generates object-detection labels (YOLO format) for road-scene images using a vision-language model (GroundingDINO) through the Autodistill framework.

It is designed for BDD100K-style data and supports both:
- full dataset annotation (train/val/test splits), and
- quick testing on a small folder (`test/`).

---

## 1) Project Purpose

Manual annotation of thousands of images is expensive and slow. This pipeline uses text prompts (class names) to automatically detect and label objects.

### Primary goals
- Automatically annotate images with bounding boxes.
- Export labels in YOLO `.txt` format.
- Validate output visually using Supervision.
- Produce a dataset ready for downstream YOLO training.

---

## 2) What the project does

### A) Annotation engine
Script: [auto_annotate.py](auto_annotate.py)

- Loads GroundingDINO via `autodistill-grounding-dino`
- Uses ontology prompts (class descriptions)
- Runs inference over dataset splits
- Writes YOLO labels per image
- Applies class-agnostic NMS to reduce duplicate boxes
- Generates `data.yaml`
- Supports resume behavior (skips already-labeled files)

### B) Visualization and QA
Script: [visualize_annotations.py](visualize_annotations.py)

- Reads labels from `test_labels/`
- Draws boxes on images from `test/`
- Saves visualized images to `visualized/`
- Generates [annotation_preview.jpg](annotation_preview.jpg)
- Prints class-wise quality statistics

## 6.2 Annotate only the small test folder

Use this when you want quick testing on `test/` only:

```bash
cd "/home/jinto/Desktop/Automatic Annotation"
source .venv/bin/activate

python3 - << 'PY'
import os, glob, cv2
from tqdm import tqdm
from autodistill.detection import CaptionOntology
from autodistill_grounding_dino import GroundingDINO

INPUT_DIR = "test"
OUTPUT_DIR = "test_labels"
os.makedirs(OUTPUT_DIR, exist_ok=True)

ontology = CaptionOntology({
    "car": "car",
    "bus": "bus",
    "traffic light": "traffic_light",
    "person": "person"
})

model = GroundingDINO(ontology=ontology, box_threshold=0.35, text_threshold=0.25)

for img_path in tqdm(sorted(glob.glob(os.path.join(INPUT_DIR, "*.jpg"))), unit="img"):
    img = cv2.imread(img_path)
    h, w = img.shape[:2]
    det = model.predict(img)
    if det is not None and len(det) > 0:
        det = det.with_nms(threshold=0.5, class_agnostic=True)

    lines = []
    if det is not None and len(det) > 0:
        for xyxy, cls_id in zip(det.xyxy, det.class_id):
            x1,y1,x2,y2 = xyxy
            cx, cy = ((x1+x2)/2)/w, ((y1+y2)/2)/h
            bw, bh = (x2-x1)/w, (y2-y1)/h
            if bw > 0 and bh > 0:
                lines.append(f"{int(cls_id)} {cx:.6f} {cy:.6f} {bw:.6f} {bh:.6f}")

    out = os.path.join(OUTPUT_DIR, os.path.splitext(os.path.basename(img_path))[0] + ".txt")
    with open(out, "w") as f:
        f.write("\n".join(lines))

print("Done. Labels written to:", OUTPUT_DIR)
PY
```

Outputs:
- [test_labels](test_labels)

## 7) Label Format

Each YOLO label file has one line per object:

`class_id center_x center_y width height`

All coordinates are normalized to `[0, 1]`.

Example:

`0 0.430494 0.493027 0.124037 0.206263`

## 9) Troubleshooting

### A) `No module named groundingdino`
- Reinstall package:
```bash
pip uninstall rf_groundingdino -y
pip install rf-groundingdino --no-build-isolation
```

### B) `python3 auto_annotate.py` exits early
- Ensure venv is active and dependencies are installed.
- Check paths in [auto_annotate.py](auto_annotate.py): `DATASET_ROOT`, `SPLITS`, class map.

### C) Visualization runs but looks outdated
- Ensure labels are regenerated in the expected folder (`test_labels/` for test visualization).
- Remove stale outputs in [visualized](visualized) and rerun visualization.

### D) CUDA extension warnings
- GroundingDINO can run without custom compiled CUDA ops, but may be slower.
- Inference still works on GPU with standard PyTorch CUDA builds.
